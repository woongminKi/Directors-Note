## 2. 각 워크스트림 핵심

1. **타임아웃 cron** — 기존 `expireOverdueAssignments`/`assignQueued` sweep을 Vercel Cron(daily, KST 03:00)으로 배선. `/api/cron/sweep-assignments` + `CRON_SECRET` 인증. **배포 중 미들웨어 307 버그 발견·수정**(아래 §5).
2. **알림** — channel-agnostic 아웃박스(`notifications`) + `after()` 즉시발송 + daily cron 재시도. 웹푸시(SW+VAPID+`push_subscriptions`+권한 UI) + PWA 설치형. 이벤트 3종(결과공개/새배정/채점완료), 문구에 점수/등급 미포함(P2). 알림톡은 stub(보류).
3. **소비자 결제** — `payment_orders` 주문/거래 + `PaymentProvider` 추상화(KakaoPay sandbox + Stub) + ready→approve 리다이렉트. pay-to-unlock(채점 후 결제→release). 가격 ₩9,900. **scored+미결제 게이트**(최종 리뷰에서 과금 구멍 발견·수정).
4. **환불** — 관리자 전액 환불(`refundOrder`), `PaymentProvider.cancel()`, 환불 시 **재잠금**(paid_at 해제 + released→scored). 최소 admin 페이지 `/payments`.
5. **정산 원장** — released 시 primary 평가자에게 **₩6,000 적립**(pending, 멱등), 환불 시 void. `evaluator_earnings`. 실패 격리. **지급(송금·원천징수)은 (c)로 차단**.

## 5. 교훈 (이번 세션에서 잡은 것)

- **미들웨어가 cron/콜백을 가로챔**: `src/proxy.ts`(Next 16 미들웨어)가 세션 없는 `/api/cron/*`를 `/login`으로 307. → `isPublic`에 `/api/cron` 추가로 해결(회귀 테스트 포함). 단위/통합이 핸들러를 직접 호출해 미들웨어를 우회하면 이런 클래스를 놓침.
- **결제 과금 구멍**: `payReady`가 scored+미결제를 안 막으면 서버액션 직접 호출로 미채점 과금/중복 과금 가능 → 게이트 추가.
- **psql 부재 환경**: 마이그레이션은 `postgres`(postgres-js)로 적용 — `sql.unsafe(file).simple()`, `max:1`(BEGIN 허용).
- **DB-gated 통합테스트는 `--no-file-parallelism` 직렬 필수**: 단일 공유 dev DB라 병렬 시 평가자 풀 간섭(cron-sweep 단언 깨짐).
- **테스트 잔여물(`rlstest+`)이 풀 오염** → cleanup이 연결오류로 죽으면 누적. 발견 시 사용자 승인 후 정리.
