# 2026-06-05 타임아웃 재배정 cron 배선 (Phase A 잔여 D-①)

Phase A 잔여 작업 D 중 **타임아웃 재배정 cron**을 구현. 기존에 admin 수동 호출만 가능했던
assignment sweep 함수를 Vercel Cron(daily)으로 배선해 자동화했다. 새 비즈니스 로직은 없고
배선·인증·관측·테스트만 추가.

## 배경

`evaluation_assignments` 는 48h SLA(`due_at = assigned_at + 48h`)를 가지며, 만료 건을
만료 처리 후 재배정하는 sweep 로직(`expireOverdueAssignments`, `assignQueued`,
`src/lib/assignment/actions.ts`)은 이미 구현돼 있었으나 스케줄러에 배선되지 않아
admin 수동 호출에만 의존했다. 이 작업이 그 갭을 닫는다.

## 만든 것

수직: **Vercel Cron(daily) → GET /api/cron/sweep-assignments (CRON_SECRET 인증) → expireOverdueAssignments() → assignQueued() → SweepResult JSON**

| 파일 | 변경 | 내용 |
|---|---|---|
| `src/lib/env.ts` | modify | server env `CRON_SECRET` 추가 (+ runtimeEnv 매핑) |
| `src/app/api/cron/sweep-assignments/route.ts` | create | cron 엔드포인트 — bearer 인증, expire→queued 호출, 결과 JSON |
| `vercel.json` | modify | `crons` 등록 (`0 18 * * *` UTC = KST 03:00 오프피크) |
| `tests/integration/_seed.ts` | modify | `seedAssignment` 에 `overdue` 옵션 추가(후방호환) |
| `tests/unit/api/cron/sweep-assignments.test.ts` | create | 인증 401/200/500 + 실행순서(expire→queued) |
| `tests/integration/assignment/cron-sweep.test.ts` | create | 실DB 만료→재배정 검증(DB-gated) |

### 핵심 설계
- **인증**: Vercel cron 이 `Authorization: Bearer ${CRON_SECRET}` 를 자동 첨부 → 라우트가 검증.
  불일치/누락 → 401(외부 무단 호출 차단). 동일 토큰으로 admin 수동 호출도 가능.
- **실행 순서**: expire 먼저(만료 primary 를 queued 로 환원 + 만료 평가자 제외 재배정),
  그다음 assignQueued(환원분 + intake 미배정분 픽업). 두 sweep 모두 멱등.
- **에러 처리**: sweep 함수가 throw 하지 않고 `{ok:false}` 반환 → 라우트는 `.ok` 검사,
  하나라도 실패 시 완료분 로깅 후 500.
- **관측성**: `SweepResult`(처리 수/재배정 수)를 JSON 응답 + 구조화 console 로그.

> 참고: Vercel cron schedule 은 UTC 기준. `0 18 * * *` = KST 03:00.
