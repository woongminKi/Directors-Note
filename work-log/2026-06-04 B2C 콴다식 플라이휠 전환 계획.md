# B2C 콴다식 플라이휠 전환 계획 (v1)

방향 확정(사장님, 2026-06-04): **콴다 literal = B2C 소비자 마켓플레이스.** 앞선 분석에서 리스크로 짚은 항목(P2/P3 reversal, 미성년 PIPA, 평가자 공급, 두 면 콜드스타트)은 **전부 수용** → 본 계획은 그것들을 "막는 사유"가 아니라 "설계·빌드 항목"으로 다룬다.

## 0. 모델
소비자(학생/학부모)가 자기 연기영상 업로드 → 자격 평가자(선생님) 풀이 루브릭 v1로 채점 → **사람 점수+피드백을 유료로** 소비자에게 제공 → 각 채점이 라벨 데이터로 축적 → AI(judge)가 shadow로 전 건 채점하며 학습 → D13(≥7/10) 통과 시 케이스 단위 인계. 사람이 day-1 제품, AI는 따라잡으면 인계.

## 1. 락된 제약 재비준 (B2C가 강제)
- **P3 (B2B seat only, 부모 paywall X) → 폐기/대체.** 소비자 건당 결제 또는 구독. 명시적 재비준 필요.
- **P2 (AI 등급 부모 비노출) → 범위 재정의.** 새 정책 라인: "사람 점수 = 구매자에게 노출(상품 그 자체) / AI shadow 점수 = 영구 internal(학습·QA 외 노출 금지)."
- **D13 게이트 → 의미 전환.** ship kill-criterion 아님(사람으로 ship). **AI 인계(graduation) 게이트**.
- **PIPA 변호사 자문 → 블로킹·확대.** 미성년 자가 업로드 = 보호자 동의 + 연령 게이트 + 생체정보 학습 동의. 더 이상 deferred 아님. **Phase A 착수와 병렬 필수 트랙.**

## 2. KEEP (엔진 재배치 — 변동 없음)
- 루브릭 v1 + 원장 채점 가이드 → 평가자 라벨링 표준/채점 UI 스키마
- calibration 하니스/metrics → inter-rater 합치도 + AI 인계 게이트
- `llm-judge.ts` + gcs-staging → shadow AI 스코어러
- embeddings → 학습 피처 + few-shot 앵커 검색
- grade-derivation 경계 → 사람·AI 공통 밴드

## 3. CHANGE (B2C 특화)
- **스키마(신규 마이그레이션 ~0014):** `users`에 소비자/평가자 role 확장(또는 별도 `evaluators`). academy/student 중심 제거 또는 평가자 소속용으로만 축소. 신규: `submissions`(uploaderUserId, videoStorageUrl, consentArtifactUrl/Version, **age/guardian 필드**, status), `evaluation_assignments`(배정 큐), `labeled_results`(평가자별 4축+holistic+rationale+rubricVersion+source=human, **movement 실컬럼화**). `ai_analyses`는 shadow 출력 저장으로 유지. `evaluatorUsed` CHECK → `human|cosine|llm_judge`.
- **RLS(0002 재작성):** academy 단일축 → 3분류 — 소비자=본인 submission/released 결과만 / 평가자=배정분만(타 평가자 라벨 비공개=합치도 보호) / training role=전체 코퍼스(`is_training_role()`). `delete_student()` → `delete_uploader()` 일반화(미성년 영상 학습 보존은 동의 전제).
- **파이프라인 역전:** AI-first → human-first. `factory.ts`/`route.ts`의 AI는 critical path에서 빠져 **백그라운드 shadow worker**로. SSE 스트리밍 진행 UI 비활성(사람-우선이라 보여줄 AI 단계 없음). `FEATURE_AI_VIDEO_ANALYSIS` 의미 반전 = AI 자동결정 허용 게이트.
- **UI:** `coach-form/form.tsx`(불릿 폼) → 평가자 루브릭 채점 UI(0~10 슬라이더 + 한국어 근거, `JUDGE_RESPONSE_SCHEMA` 바인딩). `video-upload-flow`+`upload-action`(서명 업로드) → 소비자 인테이크 재사용(경로 `${uploaderId}/${submissionId}.mp4`).

## 4. NEW (B2C 전용 — 가장 큰 신규 영역)
1. **소비자 업로드 인테이크 + 연령 게이트 + 보호자 동의 플로우** (미성년 자가 업로드의 핵심. 현재 동의는 "성인 코치가 학생에 대해" 기록 → "업로드 시 보호자 자가 동의"는 신규).
2. **두 면 마켓플레이스 + 콜드스타트:** 수요(업로더) + 공급(평가자) 동시 부트스트랩. 초기 공급은 **친구 학원망 평가자로 시드** 후 확대.
3. **평가자 채점 워크벤치** + 라우팅/배정(라운드로빈/부하분산/QA용 중복라벨).
4. **라벨 코퍼스 + 학습 export**(JSONL: videoHash/axes/grade/rubricVersion/evaluatorId) → few-shot 앵커 공급.
5. **inter-rater QA 툴**(calibration-metrics 재사용, 사람-사람 합치도).
6. **소비자 결제/환불 + 평가자 정산** + **평가 SLA**(지연 시 소비자 경험).
7. **평가자 온보딩/자격:** 알려진 6영상 calibration 테스트로 합치도 자격 부여 + 지속 QA + 보수 티어.

## 5. OBSOLETE / 재배치
academy seat 온보딩(폐기), 부모 share-link letter(소비자 결과뷰로 대체; 단 letter 생성기는 유료 결과 포맷으로 전용 가능), 스트리밍 진행 UI(비활성), cosine-as-grader(폐기, 임베딩은 피처 유지).

## 6. 단계 (B2C, 리스크 분산)
- **선결/병렬: 변호사 자문 트랙 즉시 시작** (미성년 생체정보·보호자 동의·학습데이터 권리). Phase A와 병렬, 동의 문구·연령 게이트 설계 확정 전 prod 오픈 금지.
- **Phase A** — 소비자 업로드→사람 평가→유료 결과(동의/연령 게이트 포함). 평가자 풀은 친구 학원 시드. **AI 경로 없음.** 가치 ship + 데이터 플라이휠 시작.
- **Phase B** — AI shadow 전 건 채점 + 하니스로 AI-사람 합치도 추적(few-shot·밴드 재보정 = 사람 ground-truth 확보 후).
- **Phase C** — 고신뢰 케이스 AI 자동, borderline은 사람. 합치도 regression 시 자동 강등.

## 7. 오너 결정/설계 항목 (수용했지만 설계로 처리)
1. **결제 모델**: 건당 vs 구독 vs 패키지. 환불·SLA 정책.
2. **평가자 보수/QA**: 라벨당 정액 vs inter-rater 티어, 중복라벨 비율(합치도 비용 vs 데이터 품질).
3. **미성년 보호자 동의 메커니즘**: 자가 업로드 시 누가·어떻게 동의(본인인증/보호자 연계).
4. **학습데이터 권리**: 업로더(미성년) 영상의 영구 학습 코퍼스 사용 = 명시 옵트인 + 변호사 사인오프(최고위험, 그러나 compliance 필수 — 수용 대상 아님).
5. **콜드스타트**: 초기 수요 유입 + 평가자 공급 확보 전략.
6. **AI 등급 노출 정책 확정**: 권고 = 영구 internal.
