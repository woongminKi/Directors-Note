# 2026-06-05 환불 (관리자 전액 환불, 재잠금) — D-③ (a)

소비자 결제(카카오페이) 토대 위에 **관리자(CS) 전액 환불**을 구현. pay-to-unlock 모델이라 소비자
셀프 환불은 비범위(이미 본 결과물 악용 방지). 환불 시 결과 접근을 **재잠금**(paid_at 해제 +
released→scored). 정산 원장(b)·지급/원천징수(c)는 별도/차단 사이클.

## 만든 것

| 영역 | 파일 | 내용 |
|---|---|---|
| DB | `migrations/0022_payment_orders_canceled_at.sql`, `schema.ts` | `payment_orders.canceled_at`(audit) |
| 추상화 | `payments/{types,kakaopay-provider,stub-provider}.ts` | `PaymentProvider.cancel()` (KakaoPay /cancel, Stub ok) |
| 액션 | `payments/actions.ts` | `refundOrder(orderId)`(admin), `listRefundableOrders()` |
| UI | `src/app/(admin)/payments/{page,refund-button}.tsx` | 최소 admin 환불 페이지(`/payments`) |

### 핵심 설계
- **admin/owner 전용**(`getCurrentUser().appUser.role`). 소비자 노출 없음. `(admin)` 레이아웃 `requireRole` + 액션 자체 검사 이중 보호.
- 환불 가능 = `approved`만. `canceled`는 멱등 ok, 그 외 `not_refundable`.
- cancel 성공 후에만 상태 전이 → `payment_orders` canceled + canceled_at; `submissions` paid_at=NULL(항상) + status released→scored(재잠금, RLS가 결과 다시 가림).
- cancel 실패 시 상태 불변(부분 적용 방지).
