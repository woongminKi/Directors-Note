## release-gate 술어 (release.ts)
```
if status === 'released' → { allowed: true, alreadyReleased: true }   // 멱등 no-op
if status !== 'scored'   → { allowed: false, reason: 'not_scored' }
if paidAt === null       → { allowed: false, reason: 'not_paid' }
else                     → { allowed: true, alreadyReleased: false }
```

## primary 해결 쿼리 (release-action.ts, 트랜잭션 내)
primary = `is_redundant_label=false` 배정에 대응하는 라벨. 라벨은 (submission, evaluator)
유니크이므로 배정 join 으로 1행 해결:
```
db.select({ id: labeledResults.id })
  .from(labeledResults)
  .innerJoin(evaluationAssignments,
    and(eq(ea.submissionId, lr.submissionId),
        eq(ea.evaluatorUserId, lr.evaluatorUserId)))
  .where(and(eq(lr.submissionId, submissionId),
             eq(ea.isRedundantLabel, false)))
  .limit(1)
```
이후 `UPDATE labeled_results SET is_primary=true WHERE id = <primary>` — redundant 라벨은
건드리지 않는다(소비자에게 정확히 1개 primary 만 노출).
