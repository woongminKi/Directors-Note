## submitEvaluatorScore 트랜잭션
인가: `requireEvaluator()` → safeParse → 본인 활성(status='assigned') 배정 `findFirst`(없으면 `not_assigned`). primary 여부 = `!assignment.isRedundantLabel`. `derived_grade = deriveGradeFromScores([vocal, expression, movement, examReadiness])`.

ONE 트랜잭션(`db.transaction`):
1. `labeled_results` insert — `evaluator_user_id = user.appUser.id`(클라 입력 신뢰 X), 4 점수(numeric → String), holistic_grade, derived_grade, rationale jsonb, `rubric_version = JUDGE_RUBRIC_VERSION`, `source='human'`. `onConflictDoNothing` on UNIQUE(submission_id, evaluator_user_id) → 중복 제출 멱등 no-op.
2. 해당 배정 `status='submitted'` (WHERE status='assigned').
3. primary면 `submissions.status='scored'` (WHERE status='assigned' — 멱등/안전; redundant는 제출 상태 불변).
4. `users.labels_completed += 1`.

실패 시 `{ ok:false, error:'failed' }`, 성공 시 `{ ok:true, redirectTo:'/queue', derivedGrade }`.

쓰기는 WS4(actions.ts)와 동일하게 postgres-js 직결 `db`(트랜잭션 원자성 필요, assignments 는 WS2.3 상 service-role-only). 평가자 독립성은 read 레이어에서 보존: 큐/워크벤치는 항상 본인 배정·라벨만 조회.

## primary vs redundant 판정
배정 행의 `is_redundant_label`. false → primary(소비자 노출 대상 라벨, 제출을 scored 로 전이). true → QA 이중라벨(제출 상태 불변, inter-rater 데이터). 워크벤치/큐에서 redundant 배정은 'QA' 배지로 표시.

## 서명 영상 URL
`createServiceRoleClient().storage.from(SUBMISSION_VIDEOS_BUCKET).createSignedUrl(videoStorageUrl, 3600)` — private 객체에 1시간 서명 다운로드 URL. 서버 컴포넌트에서 활성 배정 확인 후 발급, `<video src>` 로 전달. 배정 없음/영상 없음/서명 실패 시 notFound.

## 큐 fetcher 쿼리
`evaluation_assignments INNER JOIN submissions ON submissions.id = assignment.submission_id WHERE evaluator_user_id = $1 AND status='assigned' ORDER BY due_at ASC`. 반환: assignmentId/submissionId/sceneType/dueAt/isRedundantLabel. 행 라벨 = `#{id 앞8자} · {sceneType}` + 마감 시각(KST). href=`/score/{submissionId}`. 학생 이름 등 신원 정보 미노출.

## 비고/가정
- 명세는 `submitEvaluatorScore` 인가를 `requireRole(['evaluator'])` 로 적었으나, requireRole→requireAuth 가 academy_id=null 계정을 /login 으로 리다이렉트하므로 평가자(academy 없음)에 작동 불가. 명세의 "evaluator 가드 추가/확인" 지시에 따라 `requireEvaluator`(require-consumer 패턴)로 구현. 인가 의미(role='evaluator' 강제)는 동일.
- Slider shadcn 컴포넌트 부재 → number input(step 0.5, inputMode=decimal)으로 채점. 명세가 "slider or number input" 허용.
- holisticGrade Select 은 기존 base-ui Select(value/onValueChange) 사용.
