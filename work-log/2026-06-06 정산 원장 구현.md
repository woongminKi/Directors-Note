# 2026-06-06 정산 원장 (평가자 수익 적립) — D-③ (b)

소비자 결제(카카오페이) + 환불 토대 위에 **평가자 정산 원장(적립)**을 구현. released 시 primary 평가자에게
₩6,000 적립, 환불 시 void. **실제 지급(송금·3.3% 원천징수·지급명세서)은 (c)로 차단**(사업자·세무·PIPA·정산대행 선행).

## 만든 것

| 영역 | 파일 | 내용 |
|---|---|---|
| DB | `migrations/0023_evaluator_earnings.sql`, `0024_*_rls.sql`, `schema.ts` | `evaluator_earnings` 원장 + RLS(본인 SELECT) |
| config/actions | `src/lib/settlement/{config,actions}.ts` | `EVALUATOR_FEE_KRW=6000`, `accrueEarning`/`voidEarningsForSubmission`/`listEarnings` |
| 훅 | `release-action.ts`, `payments/actions.ts` | release→적립(primary), refundOrder→void (둘 다 실패 격리) |

### 핵심 설계
- **적립 트리거 = release**(is_primary 셋되는 primary 평가자에게 ₩6,000, status=pending).
- **멱등**: `UNIQUE(submission_id, evaluator_user_id)` + onConflictDoNothing → release 멱등과 정합.
- **환불 시 void**: refundOrder가 `voidEarningsForSubmission` 호출 → pending→void(voided_at). pending만 전이(paid 보호 — 미래 (c)용).
- **실패 격리**: accrue/void 실패가 release/refund를 깨지 않음(try/catch + 로그).
- 금액 서버 config(₩6,000), 행에 스냅샷. primary만 적립(QA redundant 라벨 미적립).
