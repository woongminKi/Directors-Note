# B2C Phase A 구현 명세 (사람 평가 MVP)

전제: Phase A = 사람 평가 전용 경로(AI는 request path에 없음). 소비자 업로드 → 평가자 배정 → 루브릭 v1 4축 채점 → 유료 사람 점수+피드백 제공. 각 채점은 라벨 학습 데이터로 저장. PIPA 변호사 자문은 병렬 트랙이며 **prod 오픈을 게이트**(엔지니어링은 stub/파라미터로 진행). 전략 맥락: `2026-06-04 B2C 콴다식 플라이휠 전환 계획.md`.

> 파일 경로 정정: `src/lib/forms/coach-bullet-form.ts`, `src/lib/evaluations/upload-action.ts` (폼/액션 UI는 `src/app/(coach)/evaluation/[id]/coach-form/`).

## WS1 — 데이터 모델 (`0014_b2c_submissions.sql` + schema.ts)
DDL 관례는 0001 따름(uuid PK default gen_random_uuid(), timestamptz default now(), inline CHECK enum, idx_*). 4개 신규 테이블 + RLS(WS2).

### 1.1 users role + evaluator 필드
- `users.role` DB CHECK(0001:37) + TS $type(schema.ts:41) + `src/lib/auth/current-user.ts:8-20` 모두 `('owner','coach','admin','consumer','evaluator')`로 확장.
- `users.academy_id` 현재 NOT NULL(0001:36) → **DROP NOT NULL** (소비자/플랫폼 평가자는 academy 없음). RLS가 my_academy_id() non-null 가정 제거.
- 신규 평가자 QA 컬럼(nullable): `inter_rater_score numeric(4,3)`, `labels_completed int NOT NULL DEFAULT 0`, `onboarded_at timestamptz`(6영상 calibration 통과 전 null), `evaluator_status text CHECK IN('pending','active','suspended')`.

### 1.2 submissions (소비자 업로드 진입 — student-tied evaluations 대체)
주요 컬럼: `uploader_user_id`(FK users, RESTRICT), `scene_type`, `performance_year`, `video_storage_url`(nullable, 삭제 후 NULL), `video_lifecycle_expires_at NOT NULL`, 동의 필드(`consent_artifact_url`, `consent_version`, `consent_recorded_at`), `is_minor bool NOT NULL`, `age_band CHECK IN('under14','14_18','adult')`, `guardian_relationship`, `guardian_contact`, `training_opt_in bool NOT NULL DEFAULT false`(미성년 영구 학습용 별도 옵트인=§7.4 최고위험), `status CHECK IN('queued','assigned','scored','released') DEFAULT 'queued'`, `paid_at`(WS7), `soft_deleted_at`, created/updated.
- idx: `(uploader_user_id)`, `(status)`. 일별 유니크 없음(소비자 다중 제출 허용).
- is_minor와 training_opt_in은 **의도적으로 분리**(평가 동의 ≠ 영구 학습 동의).

### 1.3 evaluation_assignments (라우팅 큐 — 현재 analog 없음)
`submission_id`(FK CASCADE), `evaluator_user_id`(FK RESTRICT), `assigned_at`, `due_at NOT NULL`(SLA), `status CHECK IN('assigned','submitted','expired','reassigned')`, `is_redundant_label bool DEFAULT false`(QA 이중라벨).
- `UNIQUE(submission_id, evaluator_user_id)` (onConflictDoNothing 레이스 세이프, 0005 패턴).
- 부분 유니크: `uq_active_primary_assignment ON (submission_id) WHERE status='assigned' AND is_redundant_label=false` (제출당 활성 primary 1개).
- idx `(evaluator_user_id, status)` (평가자 큐).

### 1.4 labeled_results (1급 라벨 데이터 — ai_analyses 아날로그, movement 실컬럼)
`submission_id`(CASCADE), `evaluator_user_id`(RESTRICT), `vocal_score/expression_score/movement_score/exam_readiness_score numeric(3,1)`(각 BETWEEN 0 AND 10 CHECK), `holistic_grade CHECK IN(A,B,C,D)`, `derived_grade CHECK IN(A,B,C,D)`(deriveGradeFromScores 4축), `rationale jsonb`(4축 한국어), `rubric_version`(=JUDGE_RUBRIC_VERSION), `source CHECK IN('human','cosine','llm_judge') DEFAULT 'human'`, created.
- `UNIQUE(submission_id, evaluator_user_id)` — submission 단독 유니크 **금지**(이중라벨 = 다른 평가자 행). 소비자 노출은 **primary**(비-redundant) 라벨. `is_primary bool` 컬럼 권장(WS6 release 시 set).

### 1.5 기존 academy/student 테이블 처분 → **dormant 유지(마이그레이션/드롭 안 함)**
academies/students/reference_videos/evaluations/ai_analyses/feedback_drafts/embeddings 전부 그대로 보존: (a) AI 엔진 + Phase B 코치 6영상 ground-truth가 evaluations+embeddings+reference_videos 의존, (b) 마이그레이션 비용 0, B2C 진입점만 없음. evaluations→submissions 데이터 이전 안 함(소유 모델 다름). delete_student() 유지, delete_uploader()는 신규(대체 아님).

## WS2 — RLS (`0015_b2c_rls.sql`, 0002 패턴 일반화)
0002 구조 그대로(ENABLE RLS, operation별 POLICY, SECURITY DEFINER + SET search_path). 축이 my_academy_id() 단일 → **3 가시성 클래스**.

### 2.1 헬퍼
- `my_role()` (users에서 role), `is_training_role()` (service_role JWT claim 기반 — **0006대로 app.* GUC 금지**이므로 코퍼스 read는 service-role 핸들러로, is_training_role은 방어선).

### 2.2 submissions
- 소비자: select `uploader_user_id=auth.uid() AND soft_deleted_at IS NULL`; insert `uploader_user_id=auth.uid() AND my_role()='consumer'`; status 전이는 service-role 핸들러(소비자 UPDATE 없음 또는 queued 취소만).
- 평가자: select `my_role()='evaluator' AND EXISTS(배정 assigned)` — 활성 배정 중에만.
- training: `is_training_role()` 전체.

### 2.3 evaluation_assignments
- 평가자: select/update `evaluator_user_id=auth.uid()` (update는 status='submitted'만). 소비자 접근 없음. insert/reassign service-role.

### 2.4 labeled_results — **평가자 독립성(핵심 신규 RLS)**
- insert(평가자): `evaluator_user_id=auth.uid() AND EXISTS(본인 assigned 배정)`.
- select(평가자): `evaluator_user_id=auth.uid()` — **본인 라벨만**. 이 단일 술어가 독립성 보장(제출 순서 무관 타 평가자 라벨 SELECT 불가).
- select(소비자): `EXISTS(submission 본인 AND status='released') AND is_primary`.
- select training: `is_training_role()` 전체(이중라벨 포함=inter-rater 데이터셋).

### 2.5 delete_uploader() RPC (right-to-be-forgotten, delete_student 0002:218 모델)
SECURITY DEFINER + search_path + auth.uid() null 거부. 가드: 대상=auth.uid() 또는 service-role. ①users PII 익명화 ②submissions video/consent NULL + soft_deleted_at=now() ③**조건부 코퍼스 보존**: labeled_results는 `training_opt_in=true`일 때만 익명 보존, 아니면 삭제(§7.4, 변호사 사인오프 BLOCK — 분기는 설계, 정책은 파라미터). grant 잠금 0011/0012대로.

### 2.6 스토리지 버킷 RLS (0009 일반화)
경로 `${uploaderId}/${submissionId}.mp4` → 정책 `(storage.foldername(name))[1]=auth.uid()::text`. **신규 버킷 `submission-videos`**(private+500MB+mp4/mov/webm/mkv, 0009 설정).

## WS3 — 소비자 인테이크 + 동의/연령 게이트
### 3.1 업로드(video-upload-flow.tsx + upload-action.ts 재사용, AI 스트리밍 제거)
- VideoUploadFlow: streaming/complete/StreamingTimeline 분기 + "분석 시작" 버튼 제거. ready 후 "제출하기"→status='queued'+라우팅(WS4). putWithProgress XHR + 드롭존 그대로.
- upload-action: 경로 `${uploaderId}/${submissionId}.mp4`, 버킷 `submission-videos`(constants.ts 신규 상수), `attachVideoToSubmission`. createServiceRoleClient 서명업로드 유지.

### 3.2 연령 게이트 + 보호자 동의(신규, 기존 동의 일반화)
기존: 성인 코치가 학생에 대해(`recordParentConsent` students/actions.ts:113, requireRole owner/admin, CURRENT_PARENT_CONSENT_VERSION). B2C: 업로드 시 자가 동의(주로 미성년). submissions에 is_minor/age_band/guardian_*/consent_version(then-current 스탬프)/consent_recorded_at/training_opt_in 캡처. `src/lib/consent/version.ts`에 `CURRENT_UPLOADER_CONSENT_VERSION` 신규. ConsentGuidanceCard 다이얼로그 + /parent-consent 페이지 패턴 재사용.

### 3.3 게이트 로직
enqueue 서버액션 거부 조건: consent_recorded_at IS NULL / (is_minor AND guardian_contact IS NULL) / video_storage_url IS NULL. (coach-form/actions.ts:60 no_consent 게이트 미러.) 통과 시만 status='queued'.

### 3.4 변호사 BLOCK 파라미터(메커니즘 설계, 법률 텍스트 stub)
- age_band 임계(under14/14_18/adult는 placeholder — 한국 PIPA 아동동의 임계가 미결), 동의/학습옵트인 문구, 보호자 검증 강도(Phase A는 guardian_contact 자가입력 stub; 본인인증은 `FEATURE_GUARDIAN_VERIFICATION` 플래그+provider 인터페이스). **`FEATURE_B2C_INTAKE_OPEN` env(default false) = 변호사 사인오프 전 마스터 스위치 off.**

## WS4 — 라우팅/배정
**권고: app-layer 서버액션 + 레이스 세이프 클레임용 소형 SECURITY DEFINER 함수.** 선택 정책(부하분산 라운드로빈, labels_completed/evaluator_status/오픈배정수 기반)은 TS(Vitest 테스트)에, 원자적 claim은 DB(0005 + onConflictDoNothing 선례).
- 선택(`assignSubmission`): 풀=`role='evaluator' AND evaluator_status='active' AND onboarded_at NOT NULL`. 전략=오픈 배정 최소 평가자, tie=assigned_at 오래된 순. Phase A 스킬매칭 없음. `due_at=now()+SLA`(파라미터).
- 원자적 claim(`claim_assignment(...)`): uq_active_primary_assignment에 onConflictDoNothing insert + status queued→assigned. 성공여부 반환(실패 시 다음 평가자 재시도).
- 이중라벨(QA): `REDUNDANT_LABEL_RATE`(예 0.15) 확률로 다른 평가자에 is_redundant_label=true 2차 배정. inter_rater_score 피드. release 안 막음.
- 타임아웃 재배정: cron이 `assigned AND due_at<now()` → expired → 재배정(타임아웃 평가자 제외). Phase A는 수동/admin sweep로 시작 가능.
- 알림: Phase A 최소 — QueueCard 폴링(queue-card.tsx:39 refetchInterval)으로 확인. push/email 보류.

## WS5 — 평가자 채점 워크벤치
### 5.1 coach-form/form.tsx → 루브릭 채점 UI
기존 5개 free-text bullet(coachBulletFormSchema, ≥2 채움) → **0-10 축 + 한국어 근거**. **JUDGE_RESPONSE_SCHEMA(judge-rubric-v1.ts:67-94)에 바인딩**: vocal/expression/movement/examReadiness(NUMBER) + rationale(4키 STRING) + holisticGrade(A|B|C|D). 축별 number/slider(0-10 step0.5) + 한국어 Textarea(KoreanCharCounter 재사용) + 밴드 가이드(A8-10/B6-7.5/C4-5.5/D0-3.5) 인라인. holisticGrade 셀렉터(계산 등급과 별개). 비디오 플레이어(video-upload-flow.tsx:170 ready video) 재사용.

### 5.2 신규 Zod + 서버액션
- `src/lib/forms/evaluator-score-form.ts`: 4축 z.number().min(0).max(10), 4 한국어 근거 필수, holisticGrade enum. (불릿폼의 "5중 2개"와 달리 4축 전부 필수.)
- `submitEvaluatorScore(submissionId, input)`: requireRole(['evaluator']), safeParse, 활성 배정 확인(RLS 2.4 + 앱 친화 에러), `derived_grade=deriveGradeFromScores([4축])`(grade-derivation.ts:40, 가변길이 OK 변경불요), labeled_results insert(rubric_version=JUDGE_RUBRIC_VERSION, source='human', rationale jsonb), 배정 status='submitted', primary면 submissions status='scored', users.labels_completed++.

### 5.3 평가자 큐/대시보드(QueueCard 재사용)
QueueCard 제네릭(title+queryKey+fetcher+poll) 그대로. fetcher=본인 `assigned` 배정+submissions 조인. QueueRow studentName→중립 라벨(submission id+sceneType+due_at). href=`/score/{submissionId}`.

## WS6 — 소비자 결과 공개
### 6.1 결과 뷰(유료 산출물) — primary labeled_results만
4 사람 축 + derived_grade(+holistic) + 4 한국어 근거. **AI 필드 절대 노출 안 함**(B2C Phase A엔 존재도 안 함, 정책=사람 노출/AI 영구 internal).
### 6.2 전이 scored→released
서버액션(admin 또는 결제 후 자동)이 status scored→released + primary labeled_results.is_primary=true. RLS 2.4가 released+is_primary일 때만 소비자 노출 → 이 전이가 release/paywall 게이트.
### 6.3 letter 생성기 — **Phase A는 raw 점수 표시, warm letter 보류**
GPT4oMiniLetterService는 옵션(나중에 사람 라벨만으로 warm 버전, 플래그). feedback_drafts 공유링크/토큰(get_parent_feedback)은 **미사용**(소비자는 인증 계정 → RLS가 접근 경로).

## WS7 — 결제(Phase A 최소)
academies.billing_status/seat_count는 B2B seat라 무관, 확장 안 함. **권고: 플래그 stub, 실 PG 보류.** `submissions.paid_at`이 단일 결제 프리미티브. `FEATURE_PAYMENT_ENABLED`(default false): false면 stub이 paid_at=now() 즉시(친구 학원 시드 무료 파일럿), true면 한국 PG(Toss/카카오페이 — 미결) webhook으로 스탬프. release는 paid_at NOT NULL 요구. 환불/정산 Phase A 제외.

## 빌드 순서
1. WS1 0014 + schema.ts (기반) → 2. WS2 RLS 0015/0016 + role 확장(current-user/require-role) → 3. WS3 인테이크(경로+동의/연령) → 4. WS4 라우팅 → 5. WS5 워크벤치 → 6. WS7 stub + WS6 release → 7. cron 재배정 + 이중라벨 QA(비차단 폴리시).

## 테스트
- 단위(Vitest): deriveGradeFromScores 4축 경계(7.25/5.75/4.25), evaluator-score-form Zod, 동의 게이트 술어(순수함수), 라우팅 선택(라운드로빈 tie), 이중라벨 샘플링(시드).
- 통합(DB): **RLS 독립성**(평가자 A가 B의 labeled_results 못 봄), claim_assignment 레이스(2 동시 클레임 → primary 1개), delete_uploader 익명화+조건부 보존, released 전 소비자 라벨 차단.
- E2E(Playwright): 업로드→동의게이트→enqueue→배정→채점→release→소비자 결과.

## BLOCKING 미결(게이트 부분 코딩 전 필수)
1. **미성년 동의 PIPA 파라미터**: age_band 임계/보호자 검증 강도/동의·옵트인 문구 → FEATURE_B2C_INTAKE_OPEN=true(prod) 차단. 엔지니어링은 stub 진행.
2. **미성년 학습데이터 권리(§7.4)**: delete 후 영구 코퍼스 보존 가부 → 2.5 보존 분기 차단. 최고위험, 변호사 사인오프.
3. **결제 provider**: Toss/카카오페이/Stripe-KR → FEATURE_PAYMENT_ENABLED=true 차단. stub 선행.
4. **평가자 소싱/온보딩 기준**: 6영상 calibration→onboarded_at, inter_rater 임계, 시드 풀 구성 → 라우팅 자격 차단.
5. **소비자 auth**: Kakao OAuth만(kakao.ts 재사용, 권고) vs 이메일/PW 추가.
6. **SLA 값 + 이중라벨 비율**: due_at 인터벌, REDUNDANT_LABEL_RATE — WS4 상수 제품 결정.
