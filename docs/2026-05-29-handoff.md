## 4. 핵심 함정 / 주의 (꼭 읽기)

1. **dev = prod (같은 Supabase).** 로컬에서 하는 DB 변경이 곧 prod 다. 조심.
2. **`db:seed-dev` 는 가드됨** — `ALLOW_DEV_SEED=1` 없으면 거부(실학생 삭제 방지). 절대 prod 데이터에 돌리지 말 것.
3. **Vercel env 푸시 시 dotenv 따옴표 함정:** `.env.local` 의 `GOOGLE_APPLICATION_CREDENTIALS_JSON` 은 작은따옴표로 감싸져 있다. `grep|sed` 로 raw 추출해 Vercel 에 넣으면 따옴표째 들어가 `JSON.parse` 깨짐. 반드시 **dotenv 로드값**으로 푸시:
   ```bash
   bun --env-file=.env.local.prod -e 'process.stdout.write(process.env.GOOGLE_APPLICATION_CREDENTIALS_JSON)' | vercel env add GOOGLE_APPLICATION_CREDENTIALS_JSON production
   ```
4. **포트:** 이 머신은 3007. 다른 머신은 3000 가능. localhost 카카오 로그인은 해당 포트가 Supabase Redirect URLs 에 있어야 함.
5. **reference 추가/삭제는 "미래" 평가에만 영향** — 과거 평가는 저장값이라 안 바뀐다. reference 가 곧 채점 기준이므로 신중히.
6. **영상분석 self-match 주의:** 시드한 reference 와 같은 영상을 학생으로 올리면 코사인 100%·A 로 나온다(당연). 실제 검증은 다른 영상으로.
7. **AI 등급은 코치 전용** (P2) — `/review` 분석 카드는 코치만, 부모 `/feedback` 엔 letter 만.

## 5. 실행 / 테스트 / 배포

```bash
# dev
PORT=3007 bun run dev    # http://localhost:3007

# E2E (테스트 DB 가 곧 prod 라 주의 — 보통 prod 에선 안 돌림)
#   bun run db:seed-dev  ← 가드됨, 파일럿 DB 엔 쓰지 말 것
bun run lint && bun run typecheck && bun run test:ci

# prod 배포 (Vercel CLI 로그인 후)
vercel deploy --prod --yes
vercel logs https://directors-note.vercel.app   # 런타임 로그
```

**영상분석 흐름 테스트 (prod 또는 localhost):**
1. 카카오 로그인 → **학생** → 동의 완료 학생 → **시작하기 (이번 달 평가)**
2. `/evaluation/[id]` 드롭존에 영상 업로드 → 미리보기 → **분석 시작** → 수직 타임라인 → `/review`
3. `/review` 상단 **AI 분석 결과**(등급/3축/매칭) + letter 편집 → 승인·공유링크
4. **평가 기준** 메뉴(`/reference`, 원장/관리자) 에서 tier 별 reference 추가
