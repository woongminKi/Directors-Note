## 9. Real product bug — student `year` schema

E2E-S1 + E2E-S3 surfaced a real bug. `z.string().min(1).max(20).optional()` paired with the form's `""` default rejected blank submissions with zod's English default (`Too small: expected string to have >=1 characters`). The friend would have hit this on their first try.

`83e20e2` fix:
- Schema: dropped `.min(1)` (data model is nullable text, no business reason for ≥1). Korean message on `.max(20)`.
- New `normalizeYear()` helper in `src/lib/students/schema.ts` — trims, maps blank/whitespace to null.
- `createStudent` + `updateStudent` route through `normalizeYear` before insert.
- Tests cover empty/whitespace/trimmed/passthrough + the existing max-length case (now asserts Korean message).
